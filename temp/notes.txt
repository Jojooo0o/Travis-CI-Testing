Travis Setup

1. Go to travis-ci.org
2. Sign in with Github
3. Add project, flip Switch (Settings)
  1) Only build when .yml existing
  2) Limit concurrent Jobs (eg cause of extern resource access)
  3) Build Branch updates
  4) Build pull request updates

  5) Auto Cancel (Only build latest commit)


4. More Options -> Trigger build
  - Current = Overview
    - Current -> Config
  - Branches = Shows all branches of repo
  - Build History
  - Pull Requests

5. create ".travis.yml" File

  (Examples according to our project)

  1) Specify language
    -> language: java
  2) Specify version
    -> jdk:
      -oraclejdk8
  3) Run Installation
    script: mvn clean install
  4) Run Tests after successful build
    after_success:
    - mvn clean test jacoco:report coveralls:report
  5) Add some notifications to stay up to date.
    slack: digital-bauhaus:dgT2SMQCbcq3J89fYje4X0fs
    email:
      on_success: never # default:change
      on_failure: change # default:always

    (See "https://docs.travis-ci.com/user/notifications/" for more information about notifications)

6. commit & push, see the magic

7. add badge to readme (Markdown)

8. commit & push, magic again

9. Be happy, travis is set up

10. A lot more possible:
(Check Documentation: "https://docs.travis-ci.com/")
